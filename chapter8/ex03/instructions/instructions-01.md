## Perform the following tasks:

1. As a team, discuss where an html `table` would be appropriate to display information in a tabular format within one of your web pages. Consider how the `table` will be used and determine which page will contain the `table`.  The `table` will need a minimum of five columns and six rows. Include at least one `rowspan` or `colspan`. Create a new page if necessary.

2. Design the `table` as a team. Use your stylesheet to format your `table` for all viewports. Determine if the `table` is suitable for a mobile viewport. If the `table` is not the best fit for the mobile viewport, hide the `table` for the mobile viewport and use an alternate method to display the information.

3. As a team, discuss what information you would like to gather from your users from an HTML `form`. Decide what `input` controls you will use and how you will style the `form` elements for mobile, tablet, and desktop viewports. 

4. As a team, design and build the `form`. Create a new page if necessary. At a minimum, update the HTML and CSS files to do the following:
* Add a `form` to the page of your choice
* Add text `input` controls, include appropriate attributes, and make at least two fields required
* Add a `select` and `textarea` element
* Add a `form` element not used within the chapter project
* Add a `label` for each form element
* Add a submit button
* Style the `form` elements for mobile, tablet, and desktop viewports

4. Use the embedded browser tools to view and the `table` and `form` for each viewport.

5. Review your files for best coding practices; ensure proper spacing and indents for improved readability. Add comments when adding new style rules.
